# Suporte

## Como obter ajuda

O MissionApp Backend é um projeto open-source mantido voluntariamente pela equipe da [FaithTech](https://faithtech.com/). Não oferecemos suporte comercial com SLA garantido, mas faremos o melhor para responder a dúvidas e problemas dentro das nossas possibilidades.

---

## Antes de pedir ajuda

Verifique se a sua dúvida já foi respondida:

- **[README](./README.md)** — visão geral do projeto e instruções de setup
- **[CONTRIBUTING.md](./CONTRIBUTING.md)** — guia completo de configuração do ambiente de desenvolvimento
- **[Architecture Decision Records](./docs/architecture/decisions/README.md)** — razões por trás das escolhas técnicas do projeto
- **[Issues abertas e fechadas](https://github.com/MissionAppFaithTech/MissionAppBackend/issues?q=is%3Aissue)** — sua dúvida pode já ter sido discutida

---

## Canais de suporte

### Bugs e solicitações de funcionalidade

Abra uma [issue no GitHub](https://github.com/MissionAppFaithTech/MissionAppBackend/issues) com:

- Descrição clara do problema ou da funcionalidade desejada
- Passos para reproduzir (para bugs)
- Comportamento esperado vs. comportamento observado
- Versão do Node.js, sistema operacional e saída de `docker compose ps` quando relevante

### Dúvidas gerais sobre o projeto

Para dúvidas que não se encaixam em uma issue de bug ou feature, entre em contato pelo email:

**missionapp.faithtech@gmail.com**

### Vulnerabilidades de segurança

**Não abra uma issue pública para relatar vulnerabilidades de segurança.** Consulte o [SECURITY.md](./SECURITY.md) para o processo de divulgação responsável.

---

## O que não suportamos

- Customizações do projeto para casos de uso externos ao MissionApp
- Dúvidas sobre tecnologias de terceiros (AdonisJS, PostgreSQL, Docker) não relacionadas a este projeto especificamente — consulte a documentação oficial de cada ferramenta
- Suporte a versões antigas ou forks não oficiais do repositório
